# Inline Related Products — محصولات مرتبط بین متن

افزونه‌ی سبک، RTL و سئو-محور ووکامرس برای نمایش «محصولات مرتبط» به‌صورت «کارت تکی»، «گرید/لیست» یا «اسلایدر» در دل متن مقاله؛ با «شورتکد اختصاصی» یا «درج خودکار بعد از هدینگ»، و کنترل کامل ریسپانسیو در سه تب دسکتاپ/تبلت/موبایل.

نسخه: 2.1.0  |  وردپرس 6.0+  |  PHP 7.4+  |  ووکامرس 7.0+  |  لایسنس: GPL-2.0+

## ۱) ویژگی‌ها

- جستجوی زنده‌ی محصول بر اساس نام یا SKU داخل ادیتور (گوتنبرگ و کلاسیک).
- سه حالت نمایش: کارت تکی، گرید/لیست، اسلایدر/کروسل.
- گروه‌بندی چند محصول در یک بلوک با یک شورتکد مشترک.
- دو جایگاه: شورتکد دستی یا درج خودکار بعد از هدینگ انتخابی.
- ریسپانسیو کامل: تنظیمات مستقل در سه تب دسکتاپ/تبلت/موبایل با ارث‌بری خودکار.
- کنترل اندازه فونت تیتر، قیمت، قیمت خط‌خورده و دکمه — به‌تفکیک هر دستگاه.
- روشن/خاموش‌کردن عکس، توضیح کوتاه، قیمت و دکمه؛ چیدمان افقی/عمودی کارت.
- سبک و بدون وابستگی: بدون jQuery و بدون کتابخانه؛ اسلایدر با CSS scroll-snap و چند خط JS.
- بارگذاری شرطی: فایل‌های CSS/JS فقط در صفحاتی که واقعاً بلوک دارند بارگذاری می‌شوند.
- سئو-محور: لینک داخلی follow، بدون آسیب به ساختار هدینگ مقاله، بدون CLS و بدون اسکیمای تکراری.
- امن: استفاده از nonce، بررسی سطح دسترسی، پاکسازی ورودی‌ها و escape خروجی‌ها.
- سازگار با HPOS ووکامرس و APIهای رسمی وردپرس/ووکامرس.

## ۳) نصب

1. پوشه‌ی `inline-related-products` را با ساختار فایل‌های بخش «ساختار فایل‌ها» بساز.
2. آن را در `wp-content/plugins/` قرار بده، یا کل پوشه را zip کرده و از مسیر «پیشخوان ← افزونه‌ها ← افزودن ← بارگذاری افزونه» نصب کن.
3. افزونه را فعال کن و مطمئن شو ووکامرس فعال است.
4. یک نوشته را ویرایش کن؛ پایین ادیتور باکس «محصولات مرتبط داخل متن» ظاهر می‌شود.

## ۵) راهنمای استفاده

### افزودن محصول
در باکس ادیتور، نام یا SKU محصول را بنویس (از ۲ حرف به بعد جستجوی زنده انجام می‌شود) و روی نتیجه کلیک کن تا به‌صورت کارت تکی اضافه شود.

### گروه‌بندی
چند «کارت تکی» را با تیک «انتخاب برای گروه» علامت بزن، حالت گروه (اسلایدر یا گرید/لیست) را انتخاب کن و دکمه‌ی «گروه‌بندی انتخاب‌شده‌ها» را بزن. کل گروه یک واحد با یک شورتکد مشترک می‌شود؛ با «تفکیک گروه» دوباره به کارت‌های تکی برمی‌گردد.

### تعیین جایگاه
- شورتکد (دستی): کد نمایش‌داده‌شده را هر جای متن که خواستی بگذار.
- درج خودکار: هدینگ موردنظر را از دراپ‌داون انتخاب کن؛ بلوک بلافاصله بعد از همان هدینگ نمایش داده می‌شود.

> نکته: لیست هدینگ‌ها از آخرین نسخه‌ی ذخیره‌شده‌ی مقاله خوانده می‌شود؛ بعد از افزودن هدینگ جدید، ابتدا پیش‌نویس را ذخیره کن.

## ۶) تنظیمات ریسپانسیو

زیر هر بلوک سه تب هست: دسکتاپ / تبلت / موبایل. تنظیمات هر تب مستقل است، اما با ارث‌بری خودکار:

    موبایل  <-  تبلت  <-  دسکتاپ

- به‌محض تغییر یک گزینه در تبلت یا موبایل، فقط همان دستگاه override می‌شود (کنار نام تب یک نقطه‌ی نارنجی ظاهر می‌شود).
- با دکمه‌ی «بازنشانی (حذف override)» مقادیر آن تب دوباره از تب بالادست ارث‌بری می‌شوند.

بریک‌پوینت‌های استاندارد:

| دستگاه  | محدوده           |
| ------- | ---------------- |
| موبایل  | تا 600px         |
| تبلت    | 601px تا 1024px  |
| دسکتاپ  | بالای 1024px     |

گزینه‌های هر تب: نمایش/عدم‌نمایش عکس، توضیح، قیمت و دکمه؛ چیدمان کارت (افقی/عمودی)؛ و فقط برای گروه‌ها: حالت نمایش (اسلایدر/گرید)، تعداد کارت اسلایدر، جهت لیست و تعداد ستون گرید (۱ تا ۶).

## ۷) تنظیمات اندازه فونت

در هر تب دستگاه، بخش «اندازه فونت (px)» با چهار کادر مجزا وجود دارد:

| عنصر            | سلکتور CSS               |
| --------------- | ------------------------ |
| تیتر            | .irp-card__title         |
| قیمت            | .irp-card__price         |
| قیمت خط‌خورده   | .irp-card__price del     |
| دکمه خرید       | .irp-card__btn           |

- محدوده‌ی مجاز: ۱۰ تا ۶۰ پیکسل.
- خالی گذاشتن هر کادر = استفاده از اندازه‌ی پیش‌فرض قالب یا ارث‌بری از تب بالادست.

## ۸) شورتکد و درج خودکار

هر بلوک یک key یکتا دارد که در باکس ادیتور نمایش داده می‌شود و به شکل زیر استفاده می‌شود:

    [irp key="irpab12cd"]

- شورتکد فقط برای بلوک‌های «دستی» خروجی می‌دهد تا با درج خودکار تکراری نشود.
- بلوک‌های «درج خودکار» نیازی به شورتکد ندارند و خودشان بعد از هدینگ انتخابی می‌آیند.

## ۹) سفارشی‌سازی و هوک‌ها

فعال‌سازی روی برگه‌ها یا پست‌تایپ سفارشی (در functions.php قالب):

    add_filter( 'irp_post_types', function ( $types ) {
        $types[] = 'page';       // برگه‌ها
        $types[] = 'product';    // یا هر پست‌تایپ دلخواه
        return $types;
    } );

- تعداد کلمات توضیح کوتاه: مقدار 18 را در متد card_html (فایل includes/class-irp-frontend.php) کم/زیاد کن.
- رنگ دکمه/کارت: کلاس‌های .irp-card__btn و .irp-card را در assets/frontend.css ویرایش کن.

## ۱۰) سئو و کارایی

- عنوان کارت‌ها با div رندر می‌شود (نه h2/h3) تا سلسله‌مراتب هدینگ مقاله دست‌نخورده بماند.
- لینک‌های داخلی follow به صفحه‌ی محصول برای internal linking.
- بدون JSON-LD تکراری و بدون CLS (ابعاد تصویر از سایز استاندارد woocommerce_thumbnail).
- بارگذاری شرطی asset‌ها و تصاویر با loading="lazy".
- بدون وابستگی خارجی؛ فقط چند کیلوبایت CSS/JS خالص.

## ۱۱) رفع اشکال

### هنگام لود یک پرش/اسکرول افقی کوتاه دیده می‌شود و بعد از حرکت کاربر درست می‌شود
چیدمان کاملاً با CSS انجام می‌شود؛ این رفتار تقریباً همیشه از افزونه‌های سرعت است، نه خودِ افزونه:

- Remove Unused CSS / Async CSS / Optimize CSS Delivery: فایل assets/frontend.css و شناسه‌ی inline-related-products را Exclude کن (مهم‌ترین علت).
- Delay JavaScript Execution: فایل assets/slider.js (هندل irp-slider) را Exclude کن.
- سپس کش افزونه‌ی سرعت و مرورگر را پاک کن.

> از نسخه‌ی 2.1.0، scroll-snap روی حالت proximity و direction: rtl صریح تنظیم شده تا پرشِ ابتدای لود کمینه شود.

### محصولی در جستجو نمی‌آید
مطمئن شو محصول منتشرشده (publish) است و ووکامرس فعال است؛ جستجو از موتور رسمی ووکامرس (نام + SKU) استفاده می‌کند.
